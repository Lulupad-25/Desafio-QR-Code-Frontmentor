## Bem-vindo! 👋

Obrigado por conferir este desafio de front-end.

Os desafios da Frontend Mentor ajudam você a melhorar suas habilidades de programação construindo projetos realistas.

**Para fazer este desafio, você precisa ter um conhecimento básico de HTML e CSS.**

## O desafio

Seu desafio é desenvolver este componente de código QR e deixá-lo o mais próximo possível do design fornecido.

Você pode usar quaisquer ferramentas que quiser para concluir o desafio. Então, se houver algo que você queira praticar, fique à vontade para experimentar.

### Quer ajuda com o desafio?

[Entre na nossa comunidade](https://www.frontendmentor.io/community) e faça perguntas no canal **#help**.

## Onde encontrar tudo

Sua tarefa é desenvolver o projeto com base nos designs dentro da pasta `/design`. Você encontrará tanto uma versão mobile quanto uma versão desktop do design.

Os designs estão em formato JPG estático. Isso significa que você precisará usar seu próprio julgamento para estilos como `font-size`, `padding` e `margin`.

Se você quiser o arquivo de design no Figma para ganhar experiência usando ferramentas profissionais e criar projetos mais precisos mais rapidamente, você pode [assinar como membro PRO](https://www.frontendmentor.io/pro).

Você encontrará todos os recursos necessários na pasta `/images`. Os arquivos já estão otimizados.

Também existe um arquivo `style-guide.md` contendo as informações necessárias, como paleta de cores e fontes.

## Usando assistentes de programação com IA

Incluímos dois arquivos para ajudar caso você esteja usando assistentes de programação com IA (como Claude, GitHub Copilot, Cursor, etc.) enquanto trabalha neste desafio:

* `AGENTS.md` — Contém instruções detalhadas para assistentes de IA sobre como ajudar você neste desafio. Ele é adaptado ao nível de dificuldade do desafio, oferecendo mais suporte para iniciantes e incentivando mais independência em desafios avançados.
* `CLAUDE.md` — Um arquivo de referência que direciona ferramentas baseadas no Claude para as instruções do `AGENTS.md`.

**Como usar:** Você não precisa fazer nada! Esses arquivos são detectados automaticamente pela maioria das ferramentas de IA para programação. A IA lerá as instruções e ajustará seu comportamento para ser uma parceira de aprendizado melhor — guiando você para soluções em vez de apenas entregar as respostas.

**Observação:** Esses arquivos foram criados para ajudar você a *aprender*, não para fazer o trabalho por você. A IA foi instruída a fazer perguntas, dar dicas e explicar conceitos em vez de escrever soluções completas.

## Construindo seu projeto

Sinta-se à vontade para usar qualquer fluxo de trabalho com o qual você se sinta confortável. Abaixo está um processo sugerido, mas você não precisa seguir exatamente estes passos:

1. Inicialize seu projeto como um repositório público no GitHub. Criar um repositório facilitará o compartilhamento do seu código com a comunidade caso precise de ajuda. Se você não souber como fazer isso, leia este recurso do Try Git.
2. Configure seu repositório para publicar seu código em um endereço web. Isso também será útil se você precisar de ajuda durante o desafio, pois poderá compartilhar a URL do projeto junto com a URL do repositório.
3. Analise os designs para começar a planejar como irá desenvolver o projeto. Este passo é importante para pensar antecipadamente nas classes CSS e criar estilos reutilizáveis.
4. Antes de adicionar estilos, estruture seu conteúdo com HTML. Escrever primeiro o HTML ajuda a focar em uma estrutura de conteúdo bem organizada.
5. Escreva os estilos base do projeto, incluindo estilos gerais de conteúdo, como `font-family` e `font-size`.
6. Comece adicionando estilos do topo da página para baixo. Só avance para a próxima seção quando estiver satisfeito com a área atual.

## Crie um `README.md` personalizado

Recomendamos fortemente substituir este `README.md` por um personalizado. Fornecemos um modelo dentro do arquivo [`README-template.md`](./README-template.md) neste código inicial.

O modelo serve como guia do que adicionar. Um `README` personalizado ajudará você a explicar seu projeto e refletir sobre o que aprendeu. Sinta-se à vontade para editar o modelo como quiser.

Depois de adicionar suas informações ao modelo, exclua este arquivo e renomeie `README-template.md` para `README.md`. Assim ele aparecerá como README principal do seu repositório.

## Enviando sua solução

Envie sua solução na plataforma para que o restante da comunidade possa vê-la. Siga o guia completo de envio de soluções para obter dicas sobre como fazer isso.

Lembre-se: se estiver buscando feedback sobre sua solução, faça perguntas ao enviá-la. Quanto mais específicas e detalhadas forem suas perguntas, maiores as chances de receber comentários úteis da comunidade.

## Compartilhando sua solução

Existem vários lugares onde você pode compartilhar sua solução:

1. Compartilhe sua página de solução no canal **#finished-projects** da comunidade.
2. Compartilhe no X (antigo Twitter) e mencione **@frontendmentor**, incluindo as URLs do repositório e do projeto online.
3. Compartilhe sua solução no LinkedIn.
4. Escreva sobre sua experiência desenvolvendo o projeto. Falar sobre seu fluxo de trabalho, escolhas técnicas e explicar seu código é uma ótima maneira de reforçar o aprendizado. Plataformas recomendadas incluem dev.to, Hashnode e CodeNewbie.

Fornecemos modelos para ajudar você a compartilhar sua solução após enviá-la na plataforma. Sinta-se à vontade para editá-los e incluir perguntas específicas ao buscar feedback.

Quanto mais específicas forem suas perguntas, maior a probabilidade de outro membro da comunidade fornecer um feedback útil.
